## 意图

本次变更用于补齐 Payments 报表文档的可发现性和基础使用说明，回应用户关于“各类报表的每个字段没有解释”的反馈。现有文档此前只有功能级简介和 Reports API 页面，缺少一份集中说明 `report_type`、报表范围与相关 API 入口的页面，也没有把套餐页和功能页稳定地引导到该入口。本 PR 新增中英文报表指南，并补充中英文 API Reference 与指南之间的互链，帮助用户先定位适用的报表类型和生成入口，再继续补充字段级字典。

## 变更摘要

- `payments/en/guides/reports.mdx`：新增英文报表指南，汇总 `report_type`、通用导出范围输入、适用场景和字段差异说明，作为报表文档入口页。
- `payments/cn/guides/reports.mdx`：新增中文报表指南，提供与英文页对应的中文入口，并修复中文文档缺少独立报表说明页的问题。
- `payments/en/guides/features.mdx`：将 Reports 功能概述改为指向新报表指南，避免只有一句话描述而缺少后续入口。
- `payments/cn/guides/features.mdx`：将“报表与分析”改为指向中文报表指南，便于中文读者继续查看报表范围和类型说明。
- `payments/en/guides/packages.mdx`：把 `Report Export` 的导航目标从 `amounts-and-balances` 改到新的报表指南，修正文档发现路径。
- `payments/cn/guides/packages.mdx`：把“报表导出”的导航目标改到中文报表指南，修正原先错误的入口链接。
- `payments/en/api-references/payment/generate-reports.mdx`：新增跳转说明，引导读者从 API Reference 进入报表指南查看 `report_types`、范围和字段差异说明。
- `payments/en/api-references/payment/list-all-reports.mdx`：新增跳转说明，补齐列表接口到报表指南的互链。
- `payments/cn/api-references/payment/generate-reports.mdx`：新增中文 API Reference 入口页，承接中文报表指南对 `Generate reports` 的本地化链接，解决中文页仍跳英文路径的问题。
- `payments/cn/api-references/payment/list-all-reports.mdx`：新增中文 API Reference 入口页，承接中文报表指南对 `List all reports` 的本地化链接，解决中文页仍跳英文路径的问题。

## 待确认事项

- 报表字段字典：已检索当前 docs 仓库中的报表指南、`payments/en/api-references/payment/generate-reports.mdx`、`payments/en/api-references/payment/list-all-reports.mdx` 以及 OpenAPI 中可见的 `report_types` 线索；在本仓库内未找到各报表导出文件的列级字段字典。在将该文档视为权威说明之前，需要人工从实际报表导出定义、CSV/XLSX 模板或导出服务代码确认每种报表的完整列集合与字段含义。
- 报表类型与列集合映射：已检索文档中列出的 `PayinWeeklyStatement`、`PayinDailyStatement`、`PayoutWeeklyStatement`、`PayoutDailyStatement`、`Fees`、`BalanceChange`、`ReconciliationSummary`、`BulkSend` 等类型，但本仓库未包含这些类型到具体导出列集合的实现映射。需要人工核实报表服务或序列化器中每个 `report_type` 实际对应的列集合，特别是 `BulkSend` 等新增类型。
- 字段业务语义和值规则：已检索现有文档与 API 页面，确认当前仅覆盖请求/响应参数，未覆盖导出业务列的业务语义、金额正负规则、手续费口径、时间字段单位或时区说明。需要人工从交易模型、记账模型和导出映射逻辑核实这些字段语义后，再补充字段级文档。
- 报表范围过滤与条件列：已检索 `start_time`、`end_time`、`report_types`、`report_export_format`、`token_ids` 等输入说明，但本仓库未说明哪些过滤条件只影响特定报表，或哪些列只在特定交易场景下出现。需要人工核实报表组装逻辑、过滤器生效范围以及 ZIP/多文件导出时的实际结构。
- 信息架构与发现路径：已检索 `payments/cn|en/guides/features.mdx`、`packages.mdx` 和相关 API 页面，并已补齐独立报表入口与互链；但在将其视为最终 IA 方案前，仍需人工确认该报表指南是否应继续演进为字段字典的长期承载页，或后续拆分为更细粒度的 reference 页面。
