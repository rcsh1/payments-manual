## 意图

本次变更针对 payments reports 文档中“导出字段说明”的证据边界进行了收紧。用户反馈指出，之前对各类报表字段的解释没有逐项回到代码库校验，而且不能只参考 OpenAPI 中的 `Report`、`ReportType`、`createReportBody` 等 API 级定义，还需要以实际导出 schema 或列定义为准。基于当前仓库内可直接验证的源码证据，本次仅保留已确认的业务范围描述，移除无法证明属于 CSV/XLSX 权威导出列清单的字段级说明，并同步修正中英文页中相关表述。

## 变更摘要

- `payments/cn/guides/reports.mdx`：移除 `Order`、`CryptoPayout`、`OffRamp`、`Refund`、`PayinCommissionFee`、`PayoutCommissionFee`、`BalanceChange`、`BulkSend` 等缺少权威导出列定义支撑的字段表，将相关 `report_type` 统一改为只记录已在仓库中核实的业务范围，并修正 `sub_status` 的格式问题与 OpenAPI/导出结果边界说明，避免把 API 元数据或本地对象字段误写为导出 schema。
- `payments/en/guides/reports.mdx`：与中文页同步收紧 reports 导出字段说明的证据边界，移除未被实际导出 schema 证明的字段级描述，统一改为基于已验证业务范围的英文说明，确保中英文内容一致。
