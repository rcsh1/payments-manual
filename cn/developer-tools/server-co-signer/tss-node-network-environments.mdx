服务端签名人需要以下网络环境以实现最佳运行：

- 不需要入站连接。
- 在安装（或升级）阶段：
  - 能够访问系统的 apt 源以安装所有必要的依赖项（仅适用于 Ubuntu 服务器）
  - 能够访问 Docker 引擎安装文件以安装 Docker 引擎
  - 能够访问 docker.io 注册表（公共 Docker 注册表）
  - 能够访问 Intel 网站以下载驱动程序（适用于 SGX 准备好的服务器）
- 当服务器正在运行时：
  - 开发环境：能够访问 ws.tss.dev.cobo.com 的 443 端口。如果您已配置 TSS Node Callback 服务器，服务端签名人还应能够访问回调服务器。
  - 生产环境：能够访问 ws.tss.cobo.com 的 443 端口。如果您已配置 TSS Node Callback 服务器，服务端签名人还应能够访问回调服务器。

- 当服务器正在运行时，您的服务器必须可以访问以下环境：
  | 环境 | 端口 | 域名 |
  |------------|------|--------|
  | 开发环境 | 443 | ws.tss.dev.cobo.com |
  | 生产环境 | 443 | ws.tss.cobo.com |
  
如果您已配置 TSS Node Callback 服务器，服务端签名人还应能够访问回调服务器。
