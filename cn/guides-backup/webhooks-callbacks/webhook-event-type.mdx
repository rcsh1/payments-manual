## Webhook 事件类型

可用的 webhook 事件类型如下：

<Info>本文所指交易为所有类型的交易，包括转账，合约调用和消息签名。</Info>

### 支付相关事件
支付相关的 Webhook 事件，请参考[事件与状态](/v2_cn/payments/status-and-events)。

### 交易类事件（充币/提币/合约调用/消息签名）

<Note>以下事件适用于所有类型的交易，包括充币、提币、合约调用以及消息签名等。</Note>

<table class="table-three-cols">
  <thead>
    <tr>
      <th>事件类型</th>
      <th>事件数据类型</th>
      <th>发生时机</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>wallets.transaction.created</code></td>
      <td><code>Transaction</code></td>
      <td>在区块链上检测到交易并生成记录。此事件不代表交易已成功。</td>
    </tr>
    <tr>
      <td><code>wallets.transaction.updated</code></td>
      <td><code>Transaction</code></td>
      <td>交易的状态或确认数发生变化。<Note>`wallets.transaction.updated` 涵盖交易的整个生命周期，包括创建、完成和失败。如果您想跟踪每个状态变化，订阅此事件即可。</Note></td>
    </tr>
    <tr>
      <td><code>wallets.transaction.succeeded</code></td>
      <td><code>Transaction</code></td>
      <td>交易已成功完成。</td>
    </tr>
    <tr>
      <td><code>wallets.transaction.failed</code></td>
      <td><code>Transaction</code></td>
      <td>交易失败。</td>
    </tr>
    <tr>
      <td><code>token.suspended.deposit</code></td>
      <td><code>SuspendedToken</code></td>
      <td>充币已暂停。</td>
    </tr>
    <tr>
      <td><code>token.suspended.withdraw</code></td>
      <td><code>SuspendedToken</code></td>
      <td>提币已暂停。</td>
    </tr>
  </tbody>
</table>
