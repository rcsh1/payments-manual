Callbacks are pre-execution approval requests. When Cobo needs your approval before executing a sensitive operation, Cobo sends a request to your registered callback endpoint and waits for your response. If your endpoint approves the request, Cobo continues the operation. If your endpoint rejects the request or does not respond in time, Cobo does not proceed.

Callbacks and webhooks serve different purposes:

- A callback asks your application to approve or reject an operation before execution.
- A webhook notifies your application after an event occurs.

| Flow | Message type | Purpose |
|:-----|:-------------|:--------|
| Bulk Send | Callback before execution | Used for approval of each operation before Cobo proceeds. |
| Payouts, refunds, pay-in orders, and other asynchronous status changes | Webhook after event | Used for status and lifecycle notifications. See [Events and status](/payments/en/guides/status-and-events). |

Currently, only **Bulk Send** triggers callback confirmation. Bulk Send can trigger callback requests for the operations that Cobo needs you to approve before execution, including smart-contract and regular transfer flows.

## Handle callback messages

Follow the steps below to handle callback messages from the Cobo payments service:

1. Create a callback endpoint.
    - Choose a server environment.
    - Define an endpoint URL.
2. Implement the handling logic on the server side.
    - Parse the callback request.
    - Verify the signature.
    - Approve or deny the operation.
3. Register the endpoint on Cobo Portal.

To create an endpoint and implement the handling logic, see [Set up a callback endpoint](/payments/en/developer-tools/set-up-callback-endpoint).

To register the endpoint, see [Set up a callback endpoint](/payments/en/developer-tools/set-up-callback-endpoint#register-the-endpoint).
