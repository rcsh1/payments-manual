This page explains Payments report types, the shared inputs used to define report scope, and where to generate or manage reports.

## Generate report scope

When you generate a report, you define the report scope with these shared inputs:

- `start_time`: Start of the reporting time range
- `end_time`: End of the reporting time range
- `report_export_format`: Export format for the generated file
- `report_types`: One or more report types to generate
- `token_ids`: Optional token filter for narrowing the report content

Use these inputs to define the time range, output format, and report scope. For detailed request and response mechanics, see the related API operations below.

## Report types and when to use them

The current Payments documentation confirms the following report types:

| `report_type` | Purpose | Common use case |
| --- | --- | --- |
| `Order` | Export summary data for pay-in orders. | Reconcile order collections and review order-level pay-in activity. |
| `OrderTransaction` | Export transaction data related to pay-in orders. | Review the transaction records associated with order-mode payments. |
| `TopUpTransaction` | Export transaction data for top-up activity. | Reconcile top-up receipts and review top-up transaction records. |
| `PayinWeeklyStatement` | Export a weekly pay-in statement covering order mode and top-up mode. | Review weekly pay-in activity across your pay-in channels. |
| `PayinDailyStatement` | Export a daily pay-in statement covering order mode and top-up mode. | Review daily pay-in activity across your pay-in channels. |
| `CryptoPayout` | Export transaction data for crypto payouts. | Reconcile crypto payout execution and review payout records. |
| `OffRamp` | Export transaction data for fiat off-ramp payouts. | Reconcile fiat payout execution and review off-ramp records. |
| `Refund` | Export transaction data for refunds. | Reconcile refund activity and review refund records. |
| `PayoutWeeklyStatement` | Export a weekly payout statement covering crypto payouts, fiat off-ramps, and refunds. | Review weekly payout activity across payout channels. |
| `PayoutDailyStatement` | Export a daily payout statement covering crypto payouts, fiat off-ramps, and refunds. | Review daily payout activity across payout channels. |
| `PayinCommissionFee` | Export pay-in commission fee data. | Reconcile fees charged for pay-in activity. |
| `PayoutCommissionFee` | Export payout commission fee data. | Reconcile fees charged for payout activity. |
| `BalanceChange` | Export account balance change data. | Review balance movements for reconciliation and account analysis. |
| `Summary` | Export summary data that combines pay-ins, payouts, and commission fees. | Review an aggregated reconciliation view across core payment activity. |
| `BulkSend` | Export transaction data for Bulk Send activity. | Reconcile batch payout execution and review Bulk Send records. |

Different report types cover different business scopes, so the exported columns can differ by `report_type`.

## Exported fields and field differences

Exported fields vary by `report_type`. The following sections explain what each exported field means and when each field appears.

### `Order`

Each row represents one pay-in order.

| Field | Description | When present |
| --- | --- | --- |
| `order_id` | Unique ID of the pay-in order. | Always present. |
| `merchant_id` | Merchant ID associated with the order. | Present when the order belongs to a merchant account. |
| `merchant_order_code` | Merchant-defined reference code for the order. | Present when you provided a merchant-side order code. |
| `psp_order_code` | Developer-defined reference code for the order. | Always present. |
| `pricing_currency` | Fiat pricing currency used for the order. | Present when a pricing currency was configured for the order. |
| `pricing_amount` | Base order amount before developer fee. | Present when a pricing amount was configured for the order. |
| `fee_amount` | Developer fee charged for the order. | Always present. |
| `payable_currency` | Token ID used for payment. | Present when the order specifies the payment token. |
| `chain_id` | Blockchain network on which the payer sends funds. | Always present. |
| `payable_amount` | Token amount the payer is expected to send. | Always present. |
| `exchange_rate` | Exchange rate used to derive `payable_amount` from the priced amount. | Present when `payable_amount` was not provided directly during order creation. |
| `amount_tolerance` | Allowed shortfall tolerance for the received amount. | Present when amount tolerance is configured for the order. |
| `receive_address` | Receiving address assigned to the order. | Always present. |
| `status` | Current pay-in order status. | Always present. |
| `received_token_amount` | Total token amount actually received for the order so far. | Always present. |
| `expired_at` | Order expiration time, in Unix timestamp seconds. | Present when the order has an expiration time. |
| `created_timestamp` | Time when the order was created, in Unix timestamp seconds. | Present when creation time is included in the export. |
| `updated_timestamp` | Time when the order was last updated, in Unix timestamp seconds. | Present when update time is included in the export. |
| `transactions` | Transactions associated with the order. | Present when the export includes linked transaction details. |
| `settlement_status` | Settlement progress for the order. | Present when settlement tracking is included for the exported row. |

### `OrderTransaction`

Each row represents one transaction associated with order-mode pay-ins.

| Field | Description | When present |
| --- | --- | --- |
| `transaction_id` | Unique ID of the transaction record. | Verified as a core transaction field; present when transaction-level rows are exported. |
| `cobo_id` | Cobo tracking ID for the transaction. | Present when Cobo tracking data is included for the transaction. |
| `request_id` | Client-provided request ID used to create the transaction request. | Present when the transaction originated from a request that carried `request_id`. |
| `wallet_id` | Wallet ID associated with the transaction source or destination. | Present when wallet information is included in the transaction record. |
| `type` | Transaction type. | Verified as a core transaction field; present when transaction-level rows are exported. |
| `status` | Current transaction status. | Verified as a core transaction field; present when transaction-level rows are exported. |
| `sub_status` | More detailed transaction sub-status. | Present when the transaction has a sub-status. |
| `failed_reason` | Failure reason for approval or signing failures. | Present only for failed transactions that include a failure reason. |
| `chain_id` | Blockchain network for the transaction. | Present when the transaction is chain-specific. |
| `token_id` | Token ID transferred in the transaction. | Present when the transaction is token-specific. |
| `source` | Transaction source object. | Present when source details are included in the export row. |
| `destination` | Transaction destination object. | Present when destination details are included in the export row. |
| `result` | Result object returned for the transaction. | Present when execution result details are included. |
| `fee` | Transaction fee object. | Present when fee details are recorded for the transaction. |
| `initiator` | Actor or API key that initiated the transaction. | Present when initiator details are recorded. |
| `initiator_type` | Type of initiator for the transaction. | Verified as a core transaction field; present when transaction-level rows are exported. |
| `confirmed_num` | Number of blockchain confirmations received. | Present when the transaction has on-chain confirmation data. |
| `confirming_threshold` | Confirmation threshold used for the transaction. | Present when confirmation threshold data is tracked. |
| `transaction_hash` | On-chain transaction hash. | Present when the transaction has been submitted on-chain. |
| `block_info` | Block information for the transaction. | Present when block details are available. |
| `raw_tx_info` | Raw transaction payload details. | Present when raw transaction data is included. |
| `replacement` | Replacement relationship for sped-up or replaced transactions. | Present when the transaction was replaced or replaced another transaction. |
| `category` | Custom categories attached to the transaction. | Present when custom categories were assigned. |
| `description` | User-defined transaction description. | Present when a description was supplied. |
| `is_loop` | Whether the transaction was processed through Cobo Loop. | Present when loop metadata is included. |
| `cobo_category` | Cobo-defined transaction categories. | Present when Cobo category data is attached. |
| `extra` | Structured business-specific extra metadata. | Present when the transaction carries extra business data. |
| `fueling_info` | Fueling information for the transaction. | Present when fueling metadata exists. |
| `created_timestamp` | Time when the transaction was created. | Verified as a core transaction field; present when transaction-level rows are exported. |
| `updated_timestamp` | Time when the transaction was last updated. | Verified as a core transaction field; present when transaction-level rows are exported. |
| `acquiring_type` | Collection mode for the pay-in transaction. | Always present for payment transaction event data. |
| `order_id` | Pay-in order ID associated with the transaction. | Present for order-mode transactions. |
| `psp_order_code` | Developer-defined order reference for the related pay-in order. | Present for order-mode transactions when the order carries this code. |
| `payer_id` | Cobo-assigned payer identifier. | Present when payer tracking is enabled for the transaction. |
| `custom_payer_id` | Developer-defined payer identifier. | Present when you provided a custom payer identifier. |
| `merchant_id` | Merchant ID associated with the transaction. | Present when the transaction belongs to a merchant account. |
| `subscription_id` | Subscription ID related to the transaction. | Present for subscription-related transactions. |
| `action_id` | Subscription action ID related to the transaction. | Present for subscription-related transactions. |
