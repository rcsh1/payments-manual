title: "Error codes and status codes"

This article explains the common error codes and HTTP status codes you may encounter when using the Cobo Payments API and how to resolve the errors.

When a request fails, the response body contains an `error_code` field (integer), an `error_message` field describing the issue, and an `error_id` field you can use to trace the request in Cobo's logs.

### Error codes

#### General API errors

| Error code | HTTP status | Description | Solution |
|---|---|---|---|
| 2006 | 400 | One or more parameters are in an invalid format or contain unsupported values. | Validate your request body against the [API reference](/payments/en/api-references/overview). Check parameter types, enum values, and all required fields. The `error_message` field usually names the specific parameter that failed validation. |
| 2010 | 429 | You have exceeded the API access rate limit. | Reduce request frequency. Implement exponential backoff with jitter before retrying. |
| 2001 | 400 | MFA verification is required before this action can be performed. | Complete MFA verification in Cobo Portal, then retry the request. |
| 2024 | 401 | API key authentication failed. | <ul><li>If the `error_message` says the API key is not registered in this environment, switch to the correct API host and use the matching API key.</li><li>If the `error_message` says the API key is not yet activated, complete the required approval flow so the key becomes active.</li><li>If the `error_message` mentions the IP whitelist, add your server's egress IP address to the key's allowlist in Cobo Portal.</li><li>If signature verification fails, check that you are signing the correct request path and payload and that the nonce matches the `Biz-Api-Nonce` header.</li></ul>For details, see [Send your first request](/payments/en/developer-tools/get-started-with-api). |
| 2025, 4001 | 403 | Forbidden access to the requested resource. | <ul><li>Check the permissions and scopes associated with your API key. Refer to [Roles and permissions](/payments/en/guides/roles-and-permissions) for details.</li><li>If the error means the resource is out of your organization, verify that the resource you are accessing belongs to the same organization as the API key.</li><li>Make sure you are using the correct environment and API key combination.</li></ul> |
| 4002 | 400 | Your organization status does not permit this action. Your developer account may be under review or not fully activated. | Verify that your developer onboarding is complete. If the error persists, your account may have been restricted — check for any notifications in Cobo Portal. |

#### Payment resource errors

Payment-specific validation failures — such as a resource not being found or a field value being rejected — all return `"error_code": 2006`. Use the `error_message` field in the response to identify the specific cause.

| Scenario | Example `error_message` | Solution |
|---|---|---|
| Merchant, wallet, or transaction not found | Examples include `Wallet not found.`, `Transaction {transaction_id} not found.`, and `Request ID {request_id} not found.` | Verify that the referenced ID exists and belongs to your organization. |
| Token value rejected | Examples include `Token not found.` and `token_id:{token_id} not found` | Check that the token ID is correct and supported by the endpoint you are calling. |
| Invalid parameter value | Examples include `Invalid parameter: {param_name}.` and `Invalid parameter: {param_name}({param_value}).` | Compare the request against the API reference and correct the invalid field value named in `error_message`. |
| Invalid amount or address | Examples include `Error: Invalid amount. Please enter a valid numerical value.` and `Invalid withdrawal address.` | Use a valid numeric amount and a destination address that matches the specified token and chain. |

### HTTP status codes

| Status code | Description | Solution |
|---|---|---|
| 200 | OK. | N/A |
| 400 | Bad request. | Check the request parameters. |
| 401 | Unauthorized. | Check whether the API key matches the current environment, whether the API signature is correct, and whether the timestamp or nonce values are valid and consistent with the signed request. |
| 403 | Forbidden. | Check the business error code in the response body first (for example, `4001`) and then verify permissions, scopes, organization ownership, and any required feature enablement. |
| 404 | Not Found. | Check the request URL. |
| 405 | Method Not Allowed. | Use a supported HTTP method. |
| 406 | Not Acceptable. | Ensure the request content format is JSON. |
| 429 | Too Many Requests. | Reduce request frequency and try again later. |
| 500 | Internal Server Error. This error can be caused by several issues. | Check your server configuration settings and retry later. If the problem persists, use the `error_id` to contact Cobo support for investigation. |
| 502 | Bad Gateway. | Check the connectivity and try again later. |
| 503 | Service Unavailable. | Try again later. |
