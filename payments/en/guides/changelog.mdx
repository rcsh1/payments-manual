## October 2025

| Product | Change description | Release date |
|------|----------|----------|
| **Payments API** | API SDK v1.25.0 released.<br/><br/>- **Breaking changes**<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- In the [Create settlement request](/payments/en/api-references/payment/create-settlement-request) endpoint, the previously deprecated `settlements.currency` and `settlements.bank_account_id` fields have now been removed. | **Payments App** | Added auto-create wallet feature, simplifying merchant configuration process and improving integration efficiency. For details, see [Create merchant](/payments/en/guides/preparation#create-merchant). Related API operations: [Create merchant](/payments/en/api-references/payment/create-merchant), [List all merchants](/payments/en/api-references/payment/list-all-merchants). | 2025-10-21 |
| **Payments API** | API SDK v1.24.0 released. Changes as follows:<br/><br/>- **Breaking changes**<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- In the [Create settlement request](/payments/en/api-references/payment/create-settlement-request) operation, the `settlements.currency` and `settlements.bank_account_id` fields are deprecated. Please use the top-level `currency` and `bank_account_id` fields to specify the fiat currency and bank account.<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- **Java SDK breaking change**: When calling the following operations with Java SDK, the `merchantId` field must be repositioned after required parameters to avoid compilation errors, as it has become optional in this version:<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- [Create/Get top-up address](/payments/en/api-references/payment/createget-top-up-address)<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- [Update top-up address](/payments/en/api-references/payment/update-top-up-address)<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- [List payers](/payments/en/api-references/payment/list-payers)<br/><br/><br/>- **New features**<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Create merchant](/payments/en/api-references/payment/create-merchant) and [List all merchants](/payments/en/api-references/payment/list-all-merchants) operations added `wallet_setup` field to specify merchant wallet configuration. The `wallet_id` field is now deprecated.<br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Estimate fees](/payments/en/api-references/payment/estimate-fees) operation: Used to calculate fees for various operations<br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Create pay-in order](/payments/en/api-references/payment/create-pay-in-order) operation added `custom_exchange_rate` field: Supports custom exchange rates<br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Create settlement request](/payments/en/api-references/payment/create-settlement-request) request body added fields:<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- `currency`: Specifies the fiat payout currency<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- `bank_account_id`: Specifies the receiving bank account<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- `remark`: Adds notes for fiat payouts<br/><br/>- **Field changes**<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- The `merchant_id` field is now optional in the following operations:<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- [Create/Get top-up address](/payments/en/api-references/payment/createget-top-up-address)<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- [Update top-up address](/payments/en/api-references/payment/update-top-up-address)<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- [List payers](/payments/en/api-references/payment/list-payers)<br/>&nbsp;&nbsp;&nbsp;&nbsp;- `SettleRequestStatus` schema added `Canceled` status: Indicates bank transfer failure<br/>&nbsp;&nbsp;&nbsp;&nbsp;- `Settlement` schema added `currency`, `received_amount_fiat`, and `bank_account` fields: Used to describe the fiat payout currency, estimated received amount, and bank account information | 2025-10-15 |
| **Payments API** |To simplify the settlement process and improve user experience, we have optimized Payments API by removing concepts and related operations such as withdrawable balance and forced sweep:<br/><br/>**Deprecated operations:**<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- Withdrawable balance query: [Get withdrawable balances](/payments/en/api-references/payment/get-withdrawable-balances)<br/>&nbsp;&nbsp;&nbsp;&nbsp;- Payment wallet balance query: [List payment wallet balances](/payments/en/api-references/payment/list-payment-wallet-balances)<br/>&nbsp;&nbsp;&nbsp;&nbsp;- Forced sweep operation: [Create forced sweep](/payments/en/api-references/payment/create-forced-sweep), [List forced sweeps](/payments/en/api-references/payment/list-forced-sweeps)<br/><br/>**Deprecated fields:**<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- `available_balance` field in [List merchant balances](/payments/en/api-references/payment/list-merchant-balances) and [Get developer balance](/payments/en/api-references/payment/get-developer-balance)<br/>&nbsp;&nbsp;&nbsp;&nbsp;- `use_dedicated_address` field in [Create pay-in order](/payments/en/api-references/payment/create-pay-in-order)<br/><br/><Warning>The above operations and fields are retained only for compatibility with older versions and should not be used in new implementations.</Warning> | 2025-10-01 |
