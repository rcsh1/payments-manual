Cobo 支付解决方案提供三种套餐，分别面向不同阶段的用户：**Starter Test**（测试体验）、**Starter Live**（自助上线）和 **Enterprise**（企业定制）。

套餐以团队（Organization）为单位开通，每个团队只能使用一种套餐。一个 Cobo Portal 账户可以创建或加入多个团队，不同团队可以分别开通不同套餐。

## 套餐概览

|  | Starter Test | Starter Live | Enterprise |
|---|---|---|---|
| **定位** | 免费测试体验，不涉及真实资金 | 自助上线，按量付费 | 企业级，销售对接 |
| **主要功能** | 测试网收付款，部分用量有上限 | 主网收付款，支持跨链，无数量限制 | 全功能，支持 Off-Ramp 对接等更多功能 |
| **KYB**（企业身份认证） | 无需 | 需要 | 需要 |
| **Fee Station 充值** | 无需 | 需要 | 需要 |
| **有效期** | 无限期 | 无限期 | 无限期 |
| **开通入口** | [Cobo 官网](https://website.cobo.com/lp/14-day-free-trial-cobo)，选择 Cobo Payments 入口，按页面指引完成注册 | 在 Starter Test 界面点击"开通正式版" | [联系销售团队](https://www.cobo.com/zh/book-demo?utm_source=product-manual&utm_medium=product&utm_campaign=pricing-plan-intro&utm_content=enterprise-contact-sales) |

## 切换套餐

切换套餐实质上是在您的账户下新建一个团队并开通新套餐。切换后，原团队及其数据保持不变，新团队使用新套餐独立运行。

- **Starter Test → Starter Live**：在界面中点击"开通正式版"，按照引导完成 KYB 认证后，系统将为您创建一个 Starter Live 套餐的团队。创建完成后需向 Fee Station 充值，才能正常使用出款等功能。
- **Starter Live → Enterprise**：请[联系我们的销售团队](https://www.cobo.com/zh/book-demo?utm_source=product-manual&utm_medium=product&utm_campaign=pricing-plan-intro&utm_content=enterprise-contact-sales)了解详情。

## 功能对比

### Cobo Portal 环境

| | Starter Test | Starter Live | Enterprise |
|---|---|---|---|
| Cobo Portal 环境 | 仅生产环境 | 仅生产环境 | 开发环境 + 生产环境 |

Starter Test 和 Starter Live 均在 **Cobo Portal 生产环境**（[portal.cobo.com](https://portal.cobo.com/login)）中使用，没有单独的开发环境。您可以先在 Starter Test 套餐下使用测试网验证主要业务流程，再开通 Starter Live 套餐正式展业。

Enterprise 套餐同时提供开发环境（[portal.dev.cobo.com](https://portal.dev.cobo.com/login)）和生产环境，客户可以先在开发环境完成集成验证，再切换至生产环境上线。具体准备步骤请参考[开发环境前置准备](/payments/cn/guides/preparation)和[生产环境前置准备](/payments/cn/guides/preparation-prod)。

### 开发者集成

所有套餐在开发者集成能力方面保持一致，包括 Developer Console、不限次数的 API 调用、Python / JavaScript / Java / Go SDK 支持，以及全量 Webhook 事件通知。详见[开发者工具](/payments/cn/developer-tools/get-started-with-api)。
