## PSP 集成示例

作为支付服务提供商（PSP），您需要为多个商户提供收款服务，并实现自动化收入分成。典型集成流程如下：

  1. 开发环境准备：完成[前置准备](/payments/cn/guides/preparation)（创建账户、配置 API Key 和 Webhook Endpoint、向 Fee Station 充值）

  2. 商户管理与资金配置：为每个下游商户[创建商户](/payments/cn/guides/merchants)，实现独立的收支管理。

  3. 接入收款功能：通过[订单模式](/payments/cn/guides/create-orders)进行收款。
  
     - 在创建订单时，通过设置开发者费用，实现与商户的自动化收入分成。
     - （可选）使用[支付链接](/payments/cn/guides/payment-link)为商户快速搭建收款页面。
 
  4. 资金管理：使用[划转资金](/payments/cn/guides/allocate-funds)功能在账户间调整分配（如补收开发者费用或处理多付、少付、晚付等异常情况）。

  5. 资金转出：支持[Off-Ramp 对接](/payments/cn/guides/fiat-payouts)和[加密货币转账](/payments/cn/guides/crypto-payouts)。Off-Ramp 对接需要预先在[转出目的地](/payments/cn/guides/destinations)中配置银行账户。

  6. 生产环境上线：测试完成后，参考[生产环境准备](/payments/cn/guides/preparation-prod)完成配置并上线

## 游戏平台集成示例

作为游戏平台，您直接服务终端玩家，主要收入来源是玩家充值。典型集成流程如下：

  1. 开发环境准备：完成[前置准备](/payments/cn/guides/preparation)（创建账户、配置 API Key 和 Webhook Endpoint、向 Fee Station 充值）

  2. 商户配置：**无需创建新商户**，直接使用系统自动创建的默认商户即可（与您的 Cobo Portal 团队名称相同）。

  3. 接入收款功能：通过[充值模式](/payments/cn/guides/get-topup-addresses)进行收款
  
     - 为每个玩家创建一个专属的充值地址
     - 玩家使用固定地址进行充值，系统自动将充值记录关联到对应收款人
     - 通过 Payments App 或 API 查询每个收款人的充值明细

  4. 资金转出：支持[Off-Ramp 对接](/payments/cn/guides/fiat-payouts)和[加密货币转账](/payments/cn/guides/crypto-payouts)。Off-Ramp 对接需要预先在[转出目的地](/payments/cn/guides/destinations)中配置银行账户。

  5. 生产环境上线：测试完成后，参考[生产环境准备](/payments/cn/guides/preparation-prod)完成配置并上线
