本文介绍如何将您收到的加密货币转账至外部加密货币地址。

## 1. 创建加密货币转账请求

您可以通过 Payments App 或 Payments API 创建加密货币转账请求。

<Tabs>
  <Tab title="Payments App" icon="pager">
    1. 登录 Cobo Portal [开发环境](https://portal.dev.cobo.com/login)或[生产环境](https://portal.cobo.com/login)。
    2. 在左侧导航栏中点击 **Apps**，然后点击 **Payments** 卡片，启动 App。
    3. 在 App 的左侧导航栏中点击**资金管理** \> **余额**。
    4. 您可以选择提取**商户余额**或**开发者余额**。在本例中，我们选择提取商户余额。点击**商户余额**卡片上的**出金**按钮。
    5. 在弹窗中，选择**发送加密货币**。

       <img
         src="/payments/cn/images/payments/crypto-payout.png"
         className="screenshot_full_screen"
         alt="加密货币转账入口"
         title=""
       />
    6. 填写转账信息，商户名称选择后不可修改，需要填写收款代币及网络、接收地址、发送金额。

       <Note>
         如果您在**转出目的地**中开启了**将转出目的地设为白名单**的开关，则仅可以转出至已录入的转出目的地。更多详情请参考[转出目的地](/payments/cn/guides/destinations)
       </Note>
    7. 自动选择资金来源：当开关开启时，您只需要填写总发送金额，系统自动根据您钱包内各代币的可用余额分配出金金额；开关关闭时，你需要自行填写各代币的出金金额。该开关默认开启，您可关闭。
    8. 按照页面提示确认并提交转账请求。
  </Tab>
  <Tab title="Payments API" icon="code">
    1. 在创建转账请求前，您可以调用 [List merchant balances](/payments/en/api-references/payment/list-merchant-balances) 或 [Get developer balance](/payments/en/api-references/payment/get-developer-balance) 分别查询商户和开发者余额，用于确定可转账的最大金额。
    2. 调用 [Create crypto address](/payments/en/api-references/payment/create-crypto-address) 录入用于转账的加密货币地址。
    3. 调用 [Create Payout ](creat)创建转账请求。

    **基础参数**

    - `request_id`：您提供的唯一请求 ID，用于跟踪转账请求
    - `source_account`付款来源账户。当从商户账户出金时，传入 Merchant code；当从开发者账户出金时，设置为 developer
    - `payout_channel`：设置为 `Crypto`，加密货币转账

    **`详情参数（payout items`**`recipient_info`**`数组）`**

    | 参数组                | 相关参数                 |
    | ------------------ | -------------------- |
    | **payout items**   | `amount`、`token_id`  |
    | **recipient_info** | `token_id`、`address` |

    **参数说明：**

    - `amount`：加密货币金额
    - `token_id`：加密货币 ID（如 `ETH_USDT`、`TRON_USDT`）
    - `crypto address`：目标加密货币地址
  </Tab>
</Tabs>

## 2. 完成二次确认、风控审核及交易签名

在创建转账请求后，基于您的提交方式、风控设置等因素，需要完成以下操作：

- 二次确认：如您通过 Payments App 进行转账，则需要在 Cobo Guard 上确认该转账请求。
- 风控审核：如果您设置了交易风控，则可能需要审核人在 Cobo Guard 上批准该转账请求。
- 交易签名：如果您使用的是 MPC 钱包，则需要移动端签名人或服务端签名人签名，方可最终完成转账操作。

## 3. 跟进转账状态

您可以订阅 `payment.settlement.status.updated` 事件，获取转账状态的实时更新。请参考 [Webhook reference](/payments/cn/guides/status-and-events) 了解每个事件的详细触发条件和返回的数据结构。

您也可以通过 Payments App 或 Payments API 主动查询转账请求状态。

<Tabs>
  <Tab title="Payments App" icon="pager">
    1. 登录 Cobo Portal [开发环境](https://portal.dev.cobo.com/login)或[生产环境](https://portal.cobo.com/login)。
    2. 在左侧导航栏中点击 **Apps**，然后点击 **Payments** 卡片，启动 App。
    3. 在 App 的左侧导航栏中点击**资金管理** \> **余额**。
    4. 在商户账户或者开发者账户详情页，**最近活动-加密货币转账**标签页查看所有转账请求的状态和详情。
  </Tab>
  <Tab title="Payments API" icon="code">
    您可以调用以下 API 操作查询转账请求状态：

    - [Get payout information](/payments/en/api-references/payment/get-settlement-request-information) - 查询单个转账请求的详细信息
    - [List all payouts](/payments/en/api-references/payment/get-settlement-request-information) - 查询所有转账请求的详细信息
  </Tab>
</Tabs>
