本文记录了 Cobo 支付解决方案的所有重要更新。

## 2025 年 12 月

| 产品 | 变更说明 | 发布时间 |
|------|----------|----------|
| **Payments App** |  - 新增[交易对手方](/payments/cn/guides/counterparties)功能，用于管理付款方信息<br/>- 新增[转出目的地](/payments/cn/guides/destinations)功能，用于集中管理资金转出时的目的地信息。 | 2025-12-03 |

## 2025 年 11 月

| 产品 | 变更说明 | 发布时间 |
|------|----------|----------|
| **Payments API** | API SDK v1.27.0 发版。<br/><br/>**不兼容变更：**<br/><br/>- Get payer balance 操作已删除。<br/>- [Get Access Token](/v2/api-references/oauth/get-access-token) 和 [Refresh Access Token](/v2/api-references/oauth/refresh-access-token) 操作已废弃。<br/><br/>**新增功能：**<br/><br/>- 新增 [Batch get exchange rates](/payments/en/api-references/payment/batch-get-exchange-rates) 操作，用于批量获取汇率。 | 2025-11-26 |
| **Payments API** | API SDK v1.26.0 发版。<br/><br/>- 新增 [Create refund link](/payments/en/api-references/payment/create-refund-link) 操作，用于创建退款链接。<br/>- `RefundStatus` Schema 新增 `AddressPending` 和 `AddressSubmitted` 状态。 | 2025-11-12 |

## 2025 年 10 月

| 产品 | 变更说明 | 发布时间 |
|------|----------|----------|
| **Payments API** | API SDK v1.25.0 发版。<br/><br/>- **不兼容变更**<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Create settlement request](/payments/en/api-references/payment/create-settlement-request) 操作中，先前已废弃的 `settlements.currency` 和 `settlements.bank_account_id` 字段现已删除。请使用顶层的 `currency` 和 `bank_account_id` 字段来指定法币币种和银行账户。<br/> &nbsp;&nbsp;&nbsp;&nbsp;<br/>- **新增功能**<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- 新增 [Create order link](/payments/en/api-references/payment/create-order-link) 操作，用于创建收款订单的付款链接。<br/> &nbsp;&nbsp;&nbsp;&nbsp;- 新增 `payment.transaction.failed` 事件。该事件会在充币交易合规扫描不通过时触发。<br/> &nbsp;&nbsp;&nbsp;&nbsp;- [Create pay-in order](/payments/en/api-references/payment/create-pay-in-order) 请求体新增 `amount_tolerance` 字段，用于指定可接受的少付金额偏差。 | 2025-10-30 |
| **Payments App** | 新增自动创建钱包功能，简化商户配置流程，提升接入效率。详情请参考[创建商户](/payments/cn/guides/preparation#创建商户)。相关 API 操作：[Create merchant](/payments/en/api-references/payment/create-merchant)、[List all merchants](/payments/en/api-references/payment/list-all-merchants)。 | 2025-10-21 |
| **Payments API** | API SDK v1.24.0 发版。主要变更如下：<br/><br/>- **不兼容变更**<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Create settlement request](/payments/en/api-references/payment/create-settlement-request) 操作中，`settlements.currency` 和 `settlements.bank_account_id` 字段已废弃。请使用顶层的 `currency` 和 `bank_account_id` 字段来指定法币币种和银行账户。<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- **Java SDK 不兼容变更**：以下操作的 `merchantId` 字段变为可选参数后，在代码中需要将其调整至必填参数之后，否则会导致编译错误：<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- [Create/Get top-up address](/payments/en/api-references/payment/createget-top-up-address)<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- [Update top-up address](/payments/en/api-references/payment/update-top-up-address)<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- [List payers](/payments/en/api-references/payment/list-payers)<br/><br/><br/>- **新增功能**<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Create merchant](/payments/en/api-references/payment/create-merchant) 以及 [List all merchants](/payments/en/api-references/payment/list-all-merchants) 操作新增 `wallet_setup` 字段，用于指定商户的钱包配置。同时废弃了 `wallet_id` 字段。<br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Estimate fees](/payments/en/api-references/payment/estimate-fees) 操作：用于预估各类操作的手续费<br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Create pay-in order](/payments/en/api-references/payment/create-pay-in-order) 操作新增 `custom_exchange_rate` 字段：支持自定义汇率<br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Create settlement request](/payments/en/api-references/payment/create-settlement-request) 请求包体新增字段：<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- `currency`：指定法币出金币种<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- `bank_account_id`：指定收款银行账户<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- `remark`：添加法币出金备注<br/><br/>- **字段变更**<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- `merchant_id` 字段在以下操作中变更为可选：<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- [Create/Get top-up address](/payments/en/api-references/payment/createget-top-up-address)<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- [Update top-up address](/payments/en/api-references/payment/update-top-up-address)<br/>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- [List payers](/payments/en/api-references/payment/list-payers)<br/>&nbsp;&nbsp;&nbsp;&nbsp;- `SettleRequestStatus` Schema 新增 `Canceled` 状态：表示银行转账失败<br/>&nbsp;&nbsp;&nbsp;&nbsp;- `Settlement` Schema 新增 `currency`、`received_amount_fiat` 和 `bank_account` 字段：用于描述法币出金的币种、预估到账金额和银行账户信息 | 2025-10-15 |
| **Payments API** |为了简化结算流程，提升用户体验，我们对 Payments API 进行了优化，取消了可结算余额、强制归集等概念和相关操作：<br/><br/>**废弃的操作：**<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- 可结算余额查询：[Get withdrawable balances](/payments/en/api-references/payment/get-withdrawable-balances)<br/>&nbsp;&nbsp;&nbsp;&nbsp;- 支付钱包余额查询：[List payment wallet balances](/payments/en/api-references/payment/list-payment-wallet-balances)<br/>&nbsp;&nbsp;&nbsp;&nbsp;- 强制归集操作：[Create forced sweep](/payments/en/api-references/payment/create-forced-sweep)、[List forced sweeps](/payments/en/api-references/payment/list-forced-sweeps)<br/><br/>**废弃的字段：**<br/><br/>&nbsp;&nbsp;&nbsp;&nbsp;- [List merchant balances](/payments/en/api-references/payment/list-merchant-balances) 和 [Get developer balance](/payments/en/api-references/payment/get-developer-balance) 中的 `available_balance` 字段<br/>&nbsp;&nbsp;&nbsp;&nbsp;- [Create pay-in order](/payments/en/api-references/payment/create-pay-in-order) 中的 `use_dedicated_address` 字段<br/><br/><Warning>上述操作及字段仅为兼容旧版本而保留，不建议在新实现中继续使用。</Warning> | 2025-10-01 |
