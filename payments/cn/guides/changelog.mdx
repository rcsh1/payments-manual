本文记录 Cobo 支付解决方案的全部重要更新与即将上线的功能，助您及时掌握产品动态与升级信息。


## 功能预告（即将发布）

<Note>以下功能即将上线，敬请期待。</Note>

| 功能 | 功能预告 |
|------|----------|
| **资金管理** | - 新增批量转出功能，支持一次性将加密货币转账至大量地址。<br/>- 全新资金转出（Payout）API 操作，转出流程更加高效，无需区分资金来源（收款/充值模式）或指定订单，只需填写金额即可完成转出。 |

| **资金管理** | (SDK v1.28.0) 新增[划转 API](/payments/cn/guides/allocate-funds) 操作，用于在不同账户之间灵活分配资金：<br/>&nbsp;&nbsp;- [Get available allocation amount](/payments/en/api-references/payment/get-available-allocation-amount)<br/>&nbsp;&nbsp;- [Create batch allocation](/payments/en/api-references/payment/create-batch-allocation)<br/>&nbsp;&nbsp;- [Get batch allocation information](/payments/en/api-references/payment/get-batch-allocation-information)<br/>&nbsp;&nbsp;- [List all batch allocations](/payments/en/api-references/payment/list-all-batch-allocations)<br/>&nbsp;&nbsp;- [List all allocation records](/payments/en/api-references/payment/list-all-allocation-records) | 2025-12-18 |
| **收款** | (SDK v1.28.0) [Create pay-in order](/payments/en/api-references/payment/create-pay-in-order) 和 [Create order link](/payments/en/api-references/payment/create-order-link) 字段简化：支持自定义应付金额，去除 `exchange_rate` 字段，并优化了部分字段命名，增强了可读性和易用性。金额字段的全局精度统一为 2 位小数，方便对账和资金管理。<br/><br/>[Create pay-in order](/payments/en/api-references/payment/create-pay-in-order)：<br/>&nbsp;&nbsp;- 新增字段：`pricing_currency`、`pricing_amount`、`payable_currency`、`payable_amount`。<br/>&nbsp;&nbsp;- 已废弃字段：`currency`、`order_amount`、`token_id`、`custom_exchange_rate`。<br/><br/>[Create order link](/payments/en/api-references/payment/create-order-link)：<br/>&nbsp;&nbsp;- 新增字段：`pricing_currency`、`pricing_amount`、`payable_currencies`、`payable_amounts`。<br/>&nbsp;&nbsp;- 已废弃字段：`currency`、`order_amount`、`token_ids`、`custom_exchange_rates`。 | 2025-12-18 |
| **支付链接** | (SDK v1.28.0) 使用 [Create order link](/payments/en/api-references/payment/create-order-link) 创建订单支付链接时，新增 `return_url` 字段，现支持配置商户原购物页 URL。在用户支付成功或遇到异常时，可跳转回原购物页面，提升用户支付体验。 | 2025-12-18 |
| **资金管理** | - 新增[划转](/payments/cn/guides/allocate-funds)功能，支持在不同账户之间手动分配资金。目前已支持通过 Payments App 操作，API 即将开放。<br/>- 新增[账户](/payments/cn/guides/amounts-and-balances#账户)概念。账户为系统账本下管理的虚拟资金主体。详见[账户与资金分配](/payments/cn/guides/amounts-and-balances)。<br/>- 简化了资金转出流程。转出时无需选择资金来源（收款或充值模式）或指定订单，只需直接填写金额，更加高效便捷。目前已支持通过 Payments App 操作，API 即将开放。详见[Off-Ramp 对接](/payments/cn/guides/fiat-payouts)和[加密货币转账](/payments/cn/guides/crypto-payouts)。 | 2025-12-12 |
| **支付链接** | [订单支付链接](/payments/cn/guides/payment-link)全新改版：<br/>- 全新视觉与交互设计<br/>- 桌面端新增支持 Binance Wallet（EVM 及 Solana 链）<br/>- 支持扫描含金额的二维码，自动填充转账金额<br/>- 第三方钱包调用优化：调用流程简化、支持推荐前一次使用的钱包 | 2025-12-12 |
| **交易对手方** | 新增[交易对手方](/payments/cn/guides/counterparties)功能，用于管理付款方信息。 目前已支持通过 Payments App 操作，API 即将开放。| 2025-12-03 |
| **转出目的地** | 新增[转出目的地](/payments/cn/guides/destinations)功能，用于集中管理资金转出时的目的地信息。 目前已支持通过 Payments App 操作，API 即将开放。| 2025-12-03 |

| **商户管理** | 新增自动创建钱包功能，简化商户配置流程，提升接入效率。详情请参考[创建商户](/payments/cn/guides/preparation#创建商户)。相关 API 操作：[Create merchant](/payments/en/api-references/payment/create-merchant)、[List all merchants](/payments/en/api-references/payment/list-all-merchants)。 | 2025-10-21 |
| **商户管理** | (SDK v1.24.0) [Create merchant](/payments/en/api-references/payment/create-merchant) 以及 [List all merchants](/payments/en/api-references/payment/list-all-merchants) 操作新增 `wallet_setup` 字段，用于指定商户的钱包配置。同时废弃了 `wallet_id` 字段。 | 2025-10-15 |
| **资金管理** | (SDK v1.24.0) - [Estimate fees](/payments/en/api-references/payment/estimate-fees) 操作：用于预估各类操作的手续费。<br/>- [Create settlement request](/payments/en/api-references/payment/create-settlement-request) 请求包体新增 `currency`、`bank_account_id`、`remark` 字段。<br/>- `SettleRequestStatus` Schema 新增 `Canceled` 状态：表示银行转账失败。<br/>- `Settlement` Schema 新增 `currency`、`received_amount_fiat` 和 `bank_account` 字段。 | 2025-10-15 |
| **收款** | (SDK v1.24.0) - [Create pay-in order](/payments/en/api-references/payment/create-pay-in-order) 操作新增 `custom_exchange_rate` 字段：支持自定义汇率。<br/>- `merchant_id` 字段在 [Create/Get top-up address](/payments/en/api-references/payment/createget-top-up-address)、[Update top-up address](/payments/en/api-references/payment/update-top-up-address)、[List payers](/payments/en/api-references/payment/list-payers) 操作中变更为可选。 | 2025-10-15 |
| **资金管理** | 为了简化结算流程，提升用户体验，我们对 Payments API 进行了优化，取消了可结算余额、强制归集等概念和相关操作：<br/><br/>**废弃的操作：**<br/>- 可结算余额查询：[Get withdrawable balances](/payments/en/api-references/payment/get-withdrawable-balances)<br/>- 支付钱包余额查询：[List payment wallet balances](/payments/en/api-references/payment/list-payment-wallet-balances)<br/>- 强制归集操作：[Create forced sweep](/payments/en/api-references/payment/create-forced-sweep)、[List forced sweeps](/payments/en/api-references/payment/list-forced-sweeps)<br/><br/>**废弃的字段：**<br/>- [List merchant balances](/payments/en/api-references/payment/list-merchant-balances) 和 [Get developer balance](/payments/en/api-references/payment/get-developer-balance) 中的 `available_balance` 字段<br/>- [Create pay-in order](/payments/en/api-references/payment/create-pay-in-order) 中的 `use_dedicated_address` 字段<br/><br/><Warning>上述操作及字段仅为兼容旧版本而保留，不建议在新实现中继续使用。</Warning> | 2025-10-01 |
