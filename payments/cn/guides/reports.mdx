本页介绍 Payments 报表类型、用于定义报表范围的通用输入，以及生成和管理报表的入口。

## 生成报表范围

生成报表时，您可以通过以下通用输入定义报表范围：

- `start_time`：报表时间范围的开始时间
- `end_time`：报表时间范围的结束时间
- `report_export_format`：导出文件的格式
- `report_types`：要生成的一个或多个报表类型
- `token_ids`：可选的代币过滤条件，用于缩小报表范围

您可以使用这些输入定义时间范围、导出格式和报表范围。有关具体的请求与响应机制，请参见下方相关 API 操作。

## 报表类型及适用场景

当前 Payments 文档已确认以下报表类型：

| `report_type` | 用途 | 常见使用场景 |
| --- | --- | --- |
| `Order` | 导出支付订单的汇总数据。 | 用于订单收款对账，以及查看订单层级的收款活动。 |
| `OrderTransaction` | 导出与支付订单相关的交易数据。 | 用于查看订单模式收款对应的交易记录。 |
| `TopUpTransaction` | 导出充值模式的交易数据。 | 用于充值收款对账，以及查看充值交易记录。 |
| `PayinWeeklyStatement` | 导出覆盖订单模式和充值模式的周度收款报表。 | 用于按周查看各类收款渠道的活动。 |
| `PayinDailyStatement` | 导出覆盖订单模式和充值模式的日报收款表。 | 用于按日查看各类收款渠道的活动。 |
| `CryptoPayout` | 导出加密货币转账的交易数据。 | 用于加密货币转账对账，以及查看转账记录。 |
| `OffRamp` | 导出法币 Off-Ramp 转出的交易数据。 | 用于法币出金对账，以及查看 Off-Ramp 记录。 |
| `Refund` | 导出退款交易数据。 | 用于退款对账，以及查看退款记录。 |
| `PayoutWeeklyStatement` | 导出覆盖加密货币转账、法币 Off-Ramp 和退款的周度转出报表。 | 用于按周查看各类转出渠道的活动。 |
| `PayoutDailyStatement` | 导出覆盖加密货币转账、法币 Off-Ramp 和退款的日报转出表。 | 用于按日查看各类转出渠道的活动。 |
| `PayinCommissionFee` | 导出收款侧手续费数据。 | 用于对账收款相关的手续费。 |
| `PayoutCommissionFee` | 导出转出侧手续费数据。 | 用于对账转出相关的手续费。 |
| `BalanceChange` | 导出账户余额变动数据。 | 用于查看余额变化，支持对账和账户分析。 |
| `Summary` | 导出汇总数据，覆盖收款、转出和手续费。 | 用于从聚合视角查看核心支付活动的对账结果。 |
| `BulkSend` | 导出 Bulk Send 的交易数据。 | 用于批量下发执行结果对账，以及查看 Bulk Send 记录。 |

不同报表类型覆盖的业务范围不同，导出的字段也可能随 `report_type` 而变化。

## 常见枚举值说明

以下枚举值来自 Payments OpenAPI 定义。它们可能出现在报表相关 API 元数据中，也可能出现在实际包含对应字段的导出结果中。

### `report_status`

| 值 | 含义 |
| --- | --- |
| `Completed` | 报表已成功生成，可供下载或查询。 |
| `Failed` | 报表生成失败。 |

### `acquiring_type`

| 值 | 含义 |
| --- | --- |
| `Order` | 订单模式。付款人按固定金额订单完成支付，适合单次购买或固定金额场景。 |
| `TopUp` | 充值模式。付款人向专属充值地址打款，适合灵活充值或按用量结算场景。 |

### `payout_channel`

| 值 | 含义 |
| --- | --- |
| `Crypto` | 通过链上加密货币转账完成转出。 |
| `OffRamp` | 通过法币出金通道转至已登记的银行账户。 |

### `refund_type`

| 值 | 含义 |
| --- | --- |
| `Merchant` | 退款金额从商户余额扣除。 |
| `Psp` | 退款金额从开发者余额扣除。 |

### `flow_direction`

| 值 | 含义 |
| --- | --- |
| `in` | 资金流入 `source_account`。 |
| `out` | 资金流出 `source_account`。 |

### `source_type`

| 值 | 含义 |
| --- | --- |
| `Init` | 初始化导致的余额变动。 |
| `OrderIn` | 订单模式收款导致的余额变动。 |
| `LatePayment` | 迟到付款导致的余额变动。 |
| `TopUp` | 充值模式收款导致的余额变动。 |
| `UnexceptedIn` | 非预期入账导致的余额变动。 |
| `RefundOut` | 退款转出导致的余额变动。 |
| `Payout` | 转出导致的余额变动。 |
| `UnexceptedOut` | 非预期出账导致的余额变动。 |
| `Allocation` | 资金分配导致的余额变动。 |
| `SettlementNetwork` | 结算网络相关操作导致的余额变动。 |
| `BulkSend` | Bulk Send 导致的余额变动。 |

### 支付订单 `status`

| 值 | 含义 |
| --- | --- |
| `Pending` | 订单已创建，但尚未检测到入账交易。 |
| `Processing` | 已检测到入账交易，订单正在处理中。 |
| `Completed` | 订单已完成，所需支付金额已收齐。 |
| `Expired` | 订单已过期，或被更新接口显式取消。 |
| `Underpaid` | 订单已过期，且曾收到付款，但金额不足。 |

### 转出 `status`

| 值 | 含义 |
| --- | --- |
| `Pending` | 转出已创建，等待处理。 |
| `Preparing` | 正在准备转出。 |
| `Transferring` | 正在向收款目标执行转账。 |
| `Completed` | 转出已成功完成。 |
| `PartiallyCompleted` | 部分转账成功，部分失败。 |
| `Failed` | 转出失败，没有成功完成的转账。 |
| `RejectedByBank` | 被收款银行拒绝，仅适用于 OffRamp。 |

### 退款 `status`

| 值 | 含义 |
| --- | --- |
| `AddressPending` | 用户已打开退款链接，但尚未提交退款地址。 |
| `AddressSubmitted` | 退款地址已提交。 |
| `Pending` | 退款单已创建，但退款交易尚未发起。 |
| `Processing` | 退款处理中，至少有一笔退款交易在执行。 |
| `Completed` | 所有退款交易均已成功完成。 |
| `PartiallyCompleted` | 部分退款交易成功，部分失败。 |
| `Failed` | 所有退款交易都失败。 |
| `PendingConfirmation` | 退款单已创建，但尚未指定 `to_address`。补齐地址后会转为 `Pending`。 |

### 交易 `status`

| 值 | 含义 |
| --- | --- |
| `Submitted` | 交易请求已提交到系统，等待进入后续处理流程。 |
| `PendingScreening` | 交易正在等待筛查。 |
| `PendingAuthorization` | 交易正在等待授权检查，这里既可能包括人工审批，也可能包括内部风控、KYT 或合规检查。 |
| `PendingSignature` | 交易已通过前置检查，正在等待签名或签名相关处理。 |
| `Broadcasting` | 交易正在广播到链上网络。 |
| `Confirming` | 交易已上链，正在等待足够的区块确认数。 |
| `Completed` | 交易已成功完成。 |
| `Failed` | 交易处理失败。 |
| `Rejected` | 交易被拒绝，没有进入成功执行路径。 |
| `Pending` | 交易处于待处理状态，通常表示尚未进入明确的执行阶段。 |

### 高频交易 `sub_status`

`sub_status` 用于解释交易具体卡在哪个细分环节。完整清单较长，下面列出 Payments 报表里最常见、最值得关注的一组：

| 值 | 含义 |
| --- | --- |
| `PendingCoboKYTCheck` | 正在等待 Cobo KYT 检查。 |
| `RejectedKYT` | 因 KYT 检查未通过而被拒绝。 |
| `PendingRiskControlCheck` | 正在等待风控检查。 |
| `PendingApproverCheck` | 正在等待审批人审批。 |
| `RejectedByCobo` | 因未满足 Cobo 合规要求而被拒绝。 |
| `PendingWaitSigner` | 正在等待签名方介入。 |
| `PendingSignerApproval` | 正在等待签名人批准。 |
| `PendingSignerProcessing` | 签名服务正在处理。 |
| `InsufficientBalance` | 可用余额不足。 |
| `InsufficientBalanceFundLocked` | 总余额可能足够，但可用余额被锁定，暂时无法发起该交易。 |
| `FailedBroadcasting` | 广播到链上网络失败。 |
| `PendingBlockConfirmations` | 已广播，正在等待链上确认数达到阈值。 |
| `FailedOnChain` | 已上链，但链上执行失败。 |
| `ReplacedByNewTransaction` | 原交易已被新交易替换，例如加速或替换 nonce 场景。 |
| `RejectedByCoboKYT` | 因 Cobo KYT 检查未通过而被拒绝。 |
