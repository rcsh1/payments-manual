本页介绍 Payments 报表类型、用于定义报表范围的通用输入，以及生成和管理报表的入口。

## 生成报表范围

生成报表时，您可以通过以下通用输入定义报表范围：

- `start_time`：报表时间范围的开始时间
- `end_time`：报表时间范围的结束时间
- `report_export_format`：导出文件的格式
- `report_types`：要生成的一个或多个报表类型
- `token_ids`：可选的代币过滤条件，用于缩小报表范围

您可以使用这些输入定义时间范围、导出格式和报表范围。有关具体的请求与响应机制，请参见下方相关 API 操作。

## 报表类型及适用场景

当前 Payments 文档已确认以下报表类型：

| `report_type` | 用途 | 常见使用场景 |
| --- | --- | --- |
| `Order` | 导出支付订单的汇总数据。 | 用于订单收款对账，以及查看订单层级的收款活动。 |
| `OrderTransaction` | 导出与支付订单相关的交易数据。 | 用于查看订单模式收款对应的交易记录。 |
| `TopUpTransaction` | 导出充值模式的交易数据。 | 用于充值收款对账，以及查看充值交易记录。 |
| `PayinWeeklyStatement` | 导出覆盖订单模式和充值模式的周度收款报表。 | 用于按周查看各类收款渠道的活动。 |
| `PayinDailyStatement` | 导出覆盖订单模式和充值模式的日报收款表。 | 用于按日查看各类收款渠道的活动。 |
| `CryptoPayout` | 导出加密货币转账的交易数据。 | 用于加密货币转账对账，以及查看转账记录。 |
| `OffRamp` | 导出法币 Off-Ramp 转出的交易数据。 | 用于法币出金对账，以及查看 Off-Ramp 记录。 |
| `Refund` | 导出退款交易数据。 | 用于退款对账，以及查看退款记录。 |
| `PayoutWeeklyStatement` | 导出覆盖加密货币转账、法币 Off-Ramp 和退款的周度转出报表。 | 用于按周查看各类转出渠道的活动。 |
| `PayoutDailyStatement` | 导出覆盖加密货币转账、法币 Off-Ramp 和退款的日报转出表。 | 用于按日查看各类转出渠道的活动。 |
| `PayinCommissionFee` | 导出收款侧手续费数据。 | 用于对账收款相关的手续费。 |
| `PayoutCommissionFee` | 导出转出侧手续费数据。 | 用于对账转出相关的手续费。 |
| `BalanceChange` | 导出账户余额变动数据。 | 用于查看余额变化，支持对账和账户分析。 |
| `Summary` | 导出汇总数据，覆盖收款、转出和手续费。 | 用于从聚合视角查看核心支付活动的对账结果。 |
| `BulkSend` | 导出 Bulk Send 的交易数据。 | 用于批量下发执行结果对账，以及查看 Bulk Send 记录。 |

不同报表类型覆盖的业务范围不同，因此导出的字段也可能随 `report_type` 而变化。

## 导出字段与字段差异

导出字段会随 `report_type` 而变化。以下内容按报表类型说明每个导出字段的含义，以及字段何时出现。

### `Order`

每一行代表一笔支付订单。

| 字段 | 说明 | 出现条件 |
| --- | --- | --- |
| `order_id` | 支付订单的唯一 ID。 | 始终出现。 |
| `merchant_id` | 与订单关联的商户 ID。 | 当订单属于商户账户时出现。 |
| `merchant_order_code` | 商户为订单定义的业务参考码。 | 当您提供了商户侧订单号时出现。 |
| `psp_order_code` | 开发者为订单定义的业务参考码。 | 始终出现。 |
| `pricing_currency` | 订单使用的计价法币币种。 | 当订单配置了计价币种时出现。 |
| `pricing_amount` | 不含开发者手续费的订单基础金额。 | 当订单配置了计价金额时出现。 |
| `fee_amount` | 订单收取的开发者手续费。 | 始终出现。 |
| `payable_currency` | 用于支付的代币 ID。 | 当订单指定了支付币种时出现。 |
| `chain_id` | 付款所在的区块链网络。 | 始终出现。 |
| `payable_amount` | 付款人应支付的代币金额。 | 始终出现。 |
| `exchange_rate` | 用于从计价金额换算出 `payable_amount` 的汇率。 | 当创建订单时未直接提供 `payable_amount` 时出现。 |
| `amount_tolerance` | 允许的收款金额偏差范围。 | 当订单配置了金额容差时出现。 |
| `receive_address` | 订单对应的收款地址。 | 始终出现。 |
| `status` | 当前支付订单状态。 | 始终出现。 |
| `received_token_amount` | 该订单当前实际已收到的代币总额。 | 始终出现。 |
| `expired_at` | 订单过期时间，Unix 时间戳（秒）。 | 当订单设置了过期时间时出现。 |
| `created_timestamp` | 订单创建时间，Unix 时间戳（秒）。 | 当导出包含创建时间时出现。 |
| `updated_timestamp` | 订单最近更新时间，Unix 时间戳（秒）。 | 当导出包含更新时间时出现。 |
| `transactions` | 与订单关联的交易列表。 | 当导出包含关联交易详情时出现。 |
| `settlement_status` | 订单对应的结算进度。 | 当导出包含结算跟踪信息时出现。 |

### `OrderTransaction`

每一行代表一笔订单模式收款交易。

| 字段 | 说明 | 出现条件 |
| --- | --- | --- |

| `cobo_id` | Cobo 用于跟踪该交易的 ID。 | 当交易包含 Cobo 跟踪信息时出现。 |
| `request_id` | 发起该交易请求时由您提供的请求 ID。 | 当该交易来源请求带有 `request_id` 时出现。 |
| `wallet_id` | 与交易来源或目标关联的钱包 ID。 | 当交易记录包含钱包信息时出现。 |
| `type` | 交易类型。 | 已在本地交易对象中确认；当导出为交易级别行时出现。 |
| `status` | 当前交易状态。 | 已在本地交易对象中确认；当导出为交易级别行时出现。 |
| `sub_status` | 更细粒度的交易子状态。 | 当交易存在子状态时出现。 |
| `failed_reason` | 审批失败或签名失败时的失败原因。 | 仅在失败交易且返回失败原因时出现。 |
| `chain_id` | 交易所在的区块链网络。 | 当交易与链有关时出现。 |
| `token_id` | 交易涉及的代币 ID。 | 当交易与代币有关时出现。 |
| `source` | 交易来源对象。 | 当导出包含来源详情时出现。 |
| `destination` | 交易目标对象。 | 当导出包含目标详情时出现。 |
| `result` | 交易执行结果对象。 | 当导出包含执行结果详情时出现。 |
| `fee` | 交易手续费对象。 | 当该交易记录了手续费详情时出现。 |
| `initiator` | 发起该交易的操作方或 API Key。 | 当记录了发起方详情时出现。 |
| `initiator_type` | 交易发起方类型。 | 已在本地交易对象中确认；当导出为交易级别行时出现。 |
| `confirmed_num` | 交易当前已获得的链上确认数。 | 当交易存在链上确认数据时出现。 |
| `confirming_threshold` | 判定交易安全所需的确认阈值。 | 当记录了确认阈值时出现。 |
| `transaction_hash` | 链上交易哈希。 | 当交易已提交到链上时出现。 |
| `block_info` | 交易对应的区块信息。 | 当区块详情可用时出现。 |
| `raw_tx_info` | 原始交易载荷信息。 | 当导出包含原始交易数据时出现。 |
| `replacement` | 被替换交易或替换关系信息。 | 当交易发生了替换关系时出现。 |
| `category` | 您为交易设置的自定义分类。 | 当该交易设置了自定义分类时出现。 |
| `description` | 您为交易填写的说明。 | 当创建时提供了说明时出现。 |
| `is_loop` | 是否通过 Cobo Loop 执行。 | 当导出包含 Loop 元数据时出现。 |
| `cobo_category` | Cobo 定义的交易分类。 | 当交易带有 Cobo 分类时出现。 |
| `extra` | 结构化业务扩展信息。 | 当交易携带扩展业务字段时出现。 |
| `fueling_info` | 交易补充燃料相关信息。 | 当存在燃料补充元数据时出现。 |
| `created_timestamp` | 交易创建时间。 | 已在本地交易对象中确认；当导出为交易级别行时出现。 |
| `updated_timestamp` | 交易最近更新时间。 | 已在本地交易对象中确认；当导出为交易级别行时出现。 |
| `acquiring_type` | 收款模式。 | 在支付交易事件数据中始终出现。 |
| `order_id` | 关联的支付订单 ID。 | 在订单模式交易中出现。 |
| `psp_order_code` | 关联支付订单的开发者业务参考码。 | 当订单携带该编码时，在订单模式交易中出现。 |
| `payer_id` | Cobo 分配的付款人标识。 | 当该交易启用了付款人追踪时出现。 |
| `custom_payer_id` | 您定义的付款人标识。 | 当您提供了自定义付款人标识时出现。 |
| `merchant_id` | 与交易关联的商户 ID。 | 当交易属于商户账户时出现。 |
| `subscription_id` | 与该交易关联的订阅 ID。 | 当交易与订阅有关时出现。 |
| `action_id` | 与该交易关联的订阅动作 ID。 | 当交易与订阅动作有关时出现。 |

| `source_account` | 发生余额变动的商户 ID 或 `developer` 账户。 | 当导出包含账户归属信息时出现。 |
| `update_time` | 余额更新事件触发时间，Unix 时间戳（秒）。 | 当导出采用账户余额更新事件表示时出现。 |

### `Summary`

每一行代表一条覆盖收款、转出和手续费的聚合汇总记录。

本仓库内的本地源码已确认 `Summary` 的业务范围，但没有暴露其 CSV/XLSX 导出文件的权威列清单。为避免记录未经验证的字段，本页当前仅记录该 `report_type` 的已验证业务范围。

### `BulkSend`

每一行代表一笔 Bulk Send 批次记录。

| 字段 | 说明 | 出现条件 |
| --- | --- | --- |
| `bulk_send_id` | Bulk Send 批次的唯一 ID。 | 始终出现。 |
| `request_id` | 创建 Bulk Send 批次时由您提供的请求 ID。 | 当创建批次时带有 `request_id` 时出现。 |
| `source_account` | 为该批次出资的商户 ID 或 `developer` 账户。 | 始终出现。 |
| `description` | 该批次的整体说明。 | 当创建批次时提供了描述时出现。 |
| `execution_mode` | 批次执行模式，例如严格执行或部分执行。 | 始终出现。 |
| `status` | 当前 Bulk Send 批次状态。 | 始终出现。 |
| `created_timestamp` | Bulk Send 批次创建时间，Unix 时间戳（秒）。 | 始终出现。 |
| `updated_timestamp` | Bulk Send 批次最近更新时间，Unix 时间戳（秒）。 | 始终出现。 |
| `commission_fee` | 该 Bulk Send 批次产生的手续费。 | 当该批次产生手续费时出现。 |
