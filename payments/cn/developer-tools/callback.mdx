Callback 是 Cobo 支付服务在执行敏感操作前向您请求确认的一种机制。当触发需要 Callback 确认的操作时，Cobo 会向您注册的 Callback Endpoint 发送请求。您的 Endpoint 必须返回批准或拒绝的响应，Cobo 才会继续执行。

Callback 与 Webhook 的区别如下：

- **Webhook** 在事件发生后通知您的 App。
- **Callback** 在操作执行前请求您的 App 批准或拒绝。

目前，仅**批量下发（Bulk Send）**会触发 Callback 确认。在批量下发过程中，无论底层使用智能合约还是普通转账，Cobo 都会对每笔操作调用您的 Callback Endpoint，以便您在批准执行前检查交易详情。

## 处理 Callback 消息

按照以下步骤处理 Cobo 支付服务发送的 Callback 消息：

1. 创建 Callback Endpoint。
    - 选择服务器环境。
    - 定义 Endpoint URL。
2. 在服务器端实现处理逻辑。
    - 解析 Callback 请求。
    - 验证签名。
    - 批准或拒绝操作。
3. 在 Cobo Portal 上注册 Endpoint。

要创建 Endpoint 并实现处理逻辑，请参阅[设置 Callback Endpoint](/payments/cn/developer-tools/set-up-callback-endpoint)。

要注册 Endpoint，请参阅[设置 Callback Endpoint](/payments/cn/developer-tools/set-up-callback-endpoint#注册-endpoint)。
