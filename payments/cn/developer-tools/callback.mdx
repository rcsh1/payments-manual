Callback 是执行前的审批请求。当 Cobo 在执行敏感操作前需要您的批准时，Cobo 会向您注册的 Callback Endpoint 发送请求并等待响应。如果您的 Endpoint 批准该请求，Cobo 会继续执行该操作。如果您的 Endpoint 拒绝该请求，或未在规定时间内响应，Cobo 不会继续执行。

Callback 和 Webhook 用途不同：

- Callback 会在操作执行前请求您的应用批准或拒绝该操作。
- Webhook 会在事件发生后通知您的应用。

| 流程 | 消息类型 | 用途 |
|:-----|:---------|:-----|
| Bulk Send | 执行前的 Callback | 用于在 Cobo 继续执行前审批每一笔操作。 |
| 资金转出、退款单、支付订单以及其他异步状态变化 | 事件后的 Webhook | 用于状态和生命周期通知。请参阅[事件与状态](/payments/cn/guides/status-and-events)。 |

目前，只有 **Bulk Send** 会触发 Callback 确认。对于 **Bulk Send** 功能，Cobo 可能会针对执行前需要您审批的操作发送 Callback 请求，包括智能合约和常规转账流程。

## 处理 Callback 消息

按照以下步骤处理 Cobo 支付服务发送的 Callback 消息：

1. 创建 Callback Endpoint。
    - 选择服务器环境。
    - 定义 Endpoint URL。
2. 在服务器端实现处理逻辑。
    - 解析 Callback 请求。
    - 验证签名。
    - 批准或拒绝操作。
3. 在 Cobo Portal 上注册 Endpoint。

要创建 Endpoint 并实现处理逻辑，请参阅[设置 Callback Endpoint](/payments/cn/developer-tools/set-up-callback-endpoint)。

要注册 Endpoint，请参阅[设置 Callback Endpoint](/payments/cn/developer-tools/set-up-callback-endpoint#注册-endpoint)。
